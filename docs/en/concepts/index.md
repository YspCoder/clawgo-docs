# Concept Overview

This section answers one question: what ClawGo currently is.

According to the latest README and config structure, it is best understood as a long-running **Agent Runtime** focused on:

- letting `main` and `subagents` split execution
- making internal collaboration visible, traceable, and recoverable
- managing prompts, tools, roles, providers, and nodes as engineering assets
